1. Introducción

El Sistema de Gestión de Cámaras es una aplicación web moderna desarrollada con el framework Laravel, diseñada para administrar, monitorear y controlar una red de cámaras de vigilancia. La aplicación proporciona una interfaz intuitiva y funcionalidades avanzadas que permiten a los administradores mantener un registro detallado de todas las cámaras instaladas en la red.

Este sistema fue creado como solución a la necesidad de automatizar el monitoreo y la gestión de múltiples cámaras IP, permitiendo verificar el estado de las cámaras en tiempo real mediante pings automáticos cada 5 minutos.

2. Descripción General del Proyecto

Este proyecto es una aplicación web de gestión de cámaras que implementa un sistema completo de CRUD (Create, Read, Update, Delete) para administrar cámaras de vigilancia.

Objetivos Principales

  • Gestión de Cámaras: Crear, editar, ver y eliminar registros de cámaras
  • Monitoreo Automático: Verificar el estado de cada cámara automáticamente cada 5 minutos
  • Búsqueda Avanzada: Filtrar cámaras por múltiples criterios (nombre, IP, ubicación, MAC, etc.)
  • Autenticación Segura: Sistema de login para proteger el acceso a los datos
  • Dashboard Informativo: Estadísticas en tiempo real del estado de las cámaras

5. Componentes Principales

Controladores

AuthController

Gestiona la autenticación de usuarios:

  • showLoginForm(): Muestra el formulario de login
  • login(): Valida credenciales y autentica al usuario
  • logout(): Cierra la sesión del usuario

CamaraController

Gestiona todas las operaciones con cámaras:

  • index(): Lista todas las cámaras con búsqueda y filtros avanzados
  • create(): Muestra formulario para crear cámara
  • store(): Guarda una nueva cámara en la base de datos
  • edit(): Muestra formulario para editar cámara
  • update(): Actualiza datos de una cámara existente
  • destroy(): Elimina una cámara de la base de datos
  • verificarEstado(): Verifica el estado de una cámara mediante ping

6. Modelos de Datos

Modelo: Camara

Representa una cámara de vigilancia en el sistema.

Campo Tipo Descripción
id Integer Identificador único (clave primaria)
nombre String(255) Nombre de la cámara
ip String Dirección IP de la cámara
nvr String(255) NVR (Network Video Recorder) asociado
canal String(255) Número o nombre del canal
mac String(17) Dirección MAC de la cámara
ubicacion String(255) Ubicación física de la cámara
estado String Estado actual (DISPONIBLE, CAIDA, ESPERA)
ultima_conexion DateTime Fecha y hora del último acceso exitoso
created_at DateTime Fecha de creación del registro
updated_at DateTime Fecha de última actualización

Modelo: User

Representa un usuario del sistema con credenciales de acceso.

7. Controladores en Detalle

CamaraController::index()

Implementa búsqueda avanzada con filtros para los siguientes campos:

  • Nombre
  • Dirección IP
  • Ubicación
  • NVR
  • Canal
  • Dirección MAC
  • Estado

Además calcula estadísticas en tiempo real:

  • Total de cámaras
  • Cámaras activas (DISPONIBLE)
  • Cámaras caídas (CAIDA)
  • Cámaras en espera (ESPERA)

CamaraController::store()

Valida los datos siguientes:

  • nombre: Requerido, máximo 255 caracteres
  • ip: Requerido, debe ser una IP válida
  • ubicacion: Requerido, máximo 255 caracteres
  • nvr, mac, canal: Opcionales

El estado inicial se establece automáticamente como "ESPERA".

8. Rutas de la Aplicación

Rutas Públicas (Sin autenticación)

Ruta Método Controlador Descripción
/login GET AuthController@showLoginForm Muestra formulario de login
/login POST AuthController@login Procesa el login

Rutas Protegidas (Requieren autenticación)

Ruta Método Controlador Descripción
/logout POST AuthController@logout Cierra la sesión
/camaras GET CamaraController@index Lista todas las cámaras
/camaras/create GET CamaraController@create Formulario para crear cámara
/camaras POST CamaraController@store Guarda nueva cámara
/camaras/{id}/edit GET CamaraController@edit Formulario para editar cámara
/camaras/{id} PUT CamaraController@update Actualiza cámara existente
/camaras/{id} DELETE CamaraController@destroy Elimina una cámara

9. Funcionalidades del Sistema

9.1 Autenticación de Usuarios

El sistema implementa un sistema de autenticación seguro basado en Laravel Sanctum. Los usuarios deben ingresar su email y contraseña para acceder a las funcionalidades del sistema.

9.2 Gestión CRUD de Cámaras

Crear Cámaras

Permite agregar nuevas cámaras al sistema con los siguientes datos:

  • Nombre identificativo
  • Dirección IP
  • NVR asociado (opcional)
  • Canal (opcional)
  • Dirección MAC (opcional)
  • Ubicación física

Ver Cámaras

Visualiza todas las cámaras registradas en una tabla con su estado actual. Incluye un dashboard con estadísticas de cámaras disponibles, caídas y en espera.

Editar Cámaras

Permite modificar cualquier información de una cámara existente excepto su estado.

Eliminar Cámaras

Permite eliminar registros de cámaras del sistema.

9.3 Búsqueda y Filtrado Avanzado

El sistema permite buscar cámaras por 7 criterios diferentes simultáneamente:

  • Nombre
  • Dirección IP
  • Ubicación
  • NVR
  • Canal
  • Dirección MAC
  • Estado

9.4 Monitoreo Automático de Estado

El sistema verifica automáticamente el estado de todas las cámaras cada 5 minutos mediante:

  • Comando Artisan: php artisan verificar:camaras
  • Auto-actualización: php artisan camaras:auto-actualizar

El estado se actualiza a "DISPONIBLE" si la cámara responde al ping, o "CAIDA" si no hay respuesta.

9.5 Dashboard Informativo

La página principal muestra estadísticas en tiempo real:

  • Total de cámaras registradas
  • Cantidad de cámaras disponibles
  • Cantidad de cámaras caídas
  • Cantidad de cámaras en espera

10. Guía de Instalación

Paso 1: Requisitos Previos

Asegúrate de tener instalado:

  • PHP 8.1+
  • Composer
  • Node.js
  • Git
  • MySQL/MariaDB

Paso 2: Clonar el Repositorio

git clone <url-del-repositorio> cd camaras-app

Paso 3: Instalar Dependencias

# Instalar dependencias PHP composer install # Instalar dependencias JavaScript npm install

Paso 4: Configurar Variables de Entorno

# Copiar archivo de configuración cp .env.example .env # Generar clave de aplicación php artisan key:generate

Paso 5: Configurar Base de Datos

Editar el archivo .env con tus credenciales MySQL:

DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=camaras_app DB_USERNAME=root DB_PASSWORD=

Paso 6: Ejecutar Migraciones

php artisan migrate

Paso 7: Compilar Activos Frontend

npm run build

Paso 8: Iniciar el Servidor

php artisan serve

La aplicación estará disponible en http://localhost:8000

11. Guía de Uso

11.1 Acceder al Sistema

  1. Abre tu navegador web
  2. Navega a http://localhost:8000/login
  3. Ingresa tu email y contraseña
  4. Haz clic en "Ingresar"

11.2 Ver Lista de Cámaras

  1. Una vez autenticado, serás redirigido a la lista de cámaras
  2. Verás una tabla con todas las cámaras registradas
  3. En la parte superior hay un dashboard con estadísticas

11.3 Crear Nueva Cámara

  1. Haz clic en el botón "Nueva Cámara"
  2. Completa los campos obligatorios (Nombre, IP, Ubicación)
  3. Opcionalmente completa los campos de NVR, Canal y MAC
  4. Haz clic en "Guardar"

11.4 Buscar y Filtrar Cámaras

  1. En la página principal, verás campos de búsqueda para:
    • Nombre
    • IP
    • Ubicación
    • NVR
    • Canal
    • MAC
    • Estado
  2. Ingresa los criterios de búsqueda deseados
  3. Los resultados se filtrarán automáticamente

11.5 Editar una Cámara

  1. Haz clic en el botón "Editar" de la cámara deseada
  2. Modifica los datos necesarios
  3. Haz clic en "Actualizar"

11.6 Eliminar una Cámara

  1. Haz clic en el botón "Eliminar" de la cámara deseada
  2. Confirma la eliminación
  3. La cámara será eliminada de la base de datos

11.7 Cerrar Sesión

  1. Haz clic en el menú de usuario (esquina superior derecha)
  2. Selecciona "Cerrar Sesión"
